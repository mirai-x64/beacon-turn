# 折り返し灯

止まれないパドルのブロック崩し。単一 HTML・Canvas 2D・外部依存なし。

`index.html` をブラウザで開けば、そのまま始まる。説明もタイトル画面も無い。

> **GitHub Pages はまだ配信できていない。**
> Pages 有効化 API がこのトークンでは 503 を返し、リポジトリの
> `default_workflow_permissions` も `read` のままで、いずれもエージェント側の
> 権限では変えられない。当面は `index.html` をローカルで直接開いて遊ぶこと。
> 配信するには Settings → Pages で main / (root) を選び、
> Settings → Actions → General → Workflow permissions を write にする。

## 操作

**スペースキー、またはタップ / クリック。それだけ。**

押した瞬間に光の帯の進行方向が反転する。加速も減速も慣性もない。
常に等速で走り続け、変えられるのは向きだけ。壁に当たれば自動で折り返す。

止まれないので、「あの位置で拾う」は「あの瞬間に折り返す」と同じ意味になる。
早く押しても遅く押しても、行き過ぎた分だけ横にずれて船影を見送る。
落としたときは、着弾点と帯の居た位置が線で残る。どちらへどれだけ
行き過ぎたのかは、そこで分かる。

## 速くなる

霧を一層晴らすごとに、帯の速度が上がる。これは難度調整ではない。

帯が一点を通過していられる時間は `帯の幅 ÷ 速度`、連打で作れる振れ幅は
`速度 × 連打間隔`。どちらも同じ比で決まるので、速度が上がると
「連打してその場に留まる」が成立しなくなる。序盤は連打で拾えるが、
終盤は狙った瞬間に一度だけ折り返すしかない。

計測値(帯の幅 110px、100ms 間隔で連打した場合の振れ幅):

| 残り霧 | 帯速 | 振れ幅 | 連打ホバー |
|---|---|---|---|
| 40 | 230 px/s | 23px | 効く |
| 20 | 632 px/s | 63px | 効く |
| 5 | 1215 px/s | 122px | **効かない** |
| 1 | 1402 px/s | 140px | **効かない** |

船影を落とすと霧が一層戻り、帯の速度もそのぶん下がる。
残機もスコアも無い代わりに、ここが唯一の揺り戻しになっている。
詰みは無く、速度が下がるところまで下がれば必ず拾い直せる。

## 題材

大津岬灯台。帯は回転灯の光、霧は沖に立ち込める層、船影は沖を行く船。
灯台守が回転を手で折り返している見立て。

## 検証

`verify.mjs` は CDP で Chrome を起動し、`requestAnimationFrame` を
合成タイムスタンプに差し替えて実時間より速く決定論的に回す測定用ハーネス。
成果物ではないので `index.html` からは参照していない。

    node verify.mjs

測っていること:

- 初手を押さずに見送ると必ず落とすこと(押す必要を文章でなく盤面で教える)
- 連打で作れる振れ幅が終盤に帯の幅を超えること(上表)
- 反応遅れ付きの自動操縦で 40 枚を踏破できること

着弾点を予測する操縦(反応遅れ 200ms)で 458 秒・落球 115 回で踏破。
一方、見えている位置へ向かうだけの素朴な操縦は残り数枚で頭打ちになり、
踏破できない。連打ホバーが終盤に効かなくなることが、そのまま出ている。
